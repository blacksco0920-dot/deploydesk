# ABCDeploy 前端页面设计规范

> 目标：让完全不懂部署的用户始终知道“代码从哪里来、现在走到哪里、下一步要做什么”。

## 1. 复用优先与唯一视觉语言

工作流工作区按以下顺序实现：

1. 使用 [FlowGram](https://github.com/bytedance/flowgram.ai) 提供画布、节点、端口、连线、选择、缩放、平移、适应视图和内部历史；不得自研第二套画布引擎。
2. 以 [Coze Studio](https://github.com/coze-dev/coze-studio) 已开源的工作区结构与交互模式作为资源栏、Inspector、画布工具栏、运行状态和节点表单的基线。Coze Studio 完整业务前端没有作为运行时依赖导入；只有能独立使用且许可、依赖和行为都适合桌面端的通用组件才直接采用。
3. 工作流中的通用控件优先采用 Coze Studio 同源的 Semi Design；ABCDeploy 现有组件只有在视觉规格和交互完全一致时才复用。当前 Semi 的 React 19 适配器、基础样式和主题令牌已在应用入口统一加载，并直接用于工作区操作按钮、画布工具、菜单和 Tooltip；表单及底层 Radix 控件必须复用同一套语义令牌和尺寸基线，不能形成第二套可见风格。
4. Radix UI 只补充上述体系缺少的底层能力，并统一套用工作区令牌；不得在用户可见层形成第二套按钮、输入框、弹窗或 Sheet 风格。
5. 只有 ABCDeploy 特有且无成熟实现的模式才新增组件，例如部署连接选择器、节点运行证据和服务器首次连接确认。

工作流工作区只允许一套视觉语言。不得把 Coze/FlowGram 画布嵌入 ABCDeploy 旧式大卡片、大按钮或旧 Sheet；原有业务能力可以保留，原有页面外观不是必须兼容的资产。

FlowGram 已以固定版本接入 ABCDeploy 的 React 19 + Vite + Tauri 技术栈，负责正式工作流画布；工作流模块按需加载，避免项目首页首次渲染加载整套画布依赖。引入和升级继续记录固定版本、许可证、打包体积与 Tauri WebView 行为。Coze Studio 完整前端包含大量智能体业务包，不整体移植；ABCDeploy 自己负责部署节点语义、Connection 绑定、任务、证据和运行状态。

HTML 原型用于验证产品结构、交互和本文高保真视觉基线，但不形成第二套正式组件库；正式实现以 `DeploymentWorkflowCanvas`、`DeploymentPathWorkspace` 和当前桌面端行为为准。

## 2. 信息架构

应用首页的侧栏只提供“所有项目”和“配置中心”。“添加项目”是所有项目页面里的主动作，不重复放在侧栏；首版没有团队协作和云端空间，不展示“个人空间”切换。进入项目工作流后隐藏整块应用级侧栏，顶部通过“返回 + 项目名”回到应用首页；编辑器只保留 Coze 式三栏工作区：

```text
线路与节点资源栏｜工作流画布｜节点 Inspector
```

顶部是项目与线路上下文、保存状态和线路主动作；底部浮动工具栏提供选择/抓手、缩放和适应视图。Inspector 可以关闭，但打开或关闭都不能重置画布位置。

高保真工作区采用固定视觉基线：顶部栏高 `52px`，左侧资源栏宽 `208px`，右侧 Inspector 打开时宽 `360px`，标准节点为 `200 × 118px`。工作区字号只使用 `12 / 14 / 16px` 三档：辅助事实为 `12px`，正文、表单和操作为 `14px`，节点名与面板标题为 `16px`。需要偏离这些尺寸时必须由窄窗口适配或可访问性要求驱动，不能由单个页面临时决定。

紫色只表示交互语义，包括主动作、选中、焦点、链接和当前工具；绿色只表示已经由事实证明的成功或就绪。默认、未选择和未运行状态使用中性色，警告和失败分别使用独立的警告色与危险色。颜色不能替代文字和图标，也不能用绿色装饰普通信息卡。

应用级侧栏和工作流资源栏不得以两块完整左栏同时出现。左侧资源栏负责“选择线路、连接或节点类型”，右侧 Inspector 负责“配置当前选中的对象”，两者可以同时存在。

- 所有项目：添加、打开和切换本地项目。
- 配置中心：维护可跨项目复用的代码平台、版本仓库和服务器连接。
- 项目首页：部署工作流画布。
- 项目运行记录：作为当前项目工作流的次级入口，查看线路执行、不可变版本、失败证据和历史版本重上线，不进入应用一级导航。
- 本机运行：作为项目次级入口，只负责本地服务快捷启停与配置检核。

应用首页不得常驻展示“本机环境可用”之类的全局结论。本地路径、服务识别和本机运行事实属于具体项目，进入项目后由“代码来源”节点或项目次级入口承接；Docker 等系统能力异常只在相关操作发生时提示。

不得再用“发布中心 / 测试版 / 正式版 / 版本管理”作为项目主导航。用户需要另一个测试或正式目标时，新增一条服务器线路。

## 3. 工作流画布

默认“小白上线模板”使用四类节点：

```text
代码来源 ──> 版本构建 ──> 版本存储 ──> 部署运行
```

- 采用 FlowGram/Coze Studio 的画布行为：点阵背景、节点、端口、连接线、选中态、缩放、平移、适应视图和线路工具栏。
- 拖动节点后自动保存该线路的节点位置；线路之间不共享布局。进入或切换线路时先恢复布局，再自动适应视图，保证全部节点处于可见区域。
- 节点类型、Provider、Connection、节点实例和状态分层展示。节点类型名称固定，Provider 与 Connection 是辅助信息，运行状态不能替代任何一层名称。
- 首版每条线路固定且恰好展示“代码来源、版本构建、版本存储、部署运行”四个 NodeInstance。默认线路由模板自动创建，不提供手工添加、删除、连线或服务器分支。
- NodeType 可跨线路重复实例化，Connection 可跨项目选择；NodeInstance 的配置和运行状态保持线路隔离。
- 首版不提供条件、循环、审批和通知等通用自动化节点，也不要求首次用户从空画布手动连线。
- 当前执行位置通过节点与连接线状态表达，不用整页步骤条。
- 测试、演示或其他服务器目标是独立完整线路，不是服务器分支，也不是强制晋级流程。
- 工作区一次只展示当前选中线路的一张完整流程图；多条线路通过左侧列表切换，不把多个服务器目标、不同线路节点或连线拼进同一张图。

左侧资源栏只展示线路、当前线路节点类型和可绑定连接，不放置应用级“配置中心”入口；管理全局连接时先返回应用首页。连接视图与节点候选只负责当前线路引用，不提供全局删除、批量编辑或影响范围管理。画布工具栏包含选择/抓手、缩放和适应视图。线路切换、新增线路与当前唯一主动作位于稳定工作区位置；每条线路在列表中直接提供清晰的重命名和删除入口，删除进入二次确认。允许删除最后一条线路；删除后进入无线路空状态，主动作是“创建上线线路”，重新进入项目不得擅自恢复默认线路。运行记录使用顶部工具图标进入，并通过无障碍名称和悬停提示说明含义。

“全部就绪 / 首次配置 / 授权失效 / 正在上线”属于原型测试场景，不是正式产品里的状态切换器。正式实现由系统事实自动决定页面状态，用户不能手动伪造部署状态。

## 4. 节点 Inspector

点击节点打开 Coze 式右侧常驻 Inspector，不跳页、不丢画布位置。窄窗口时 Inspector 可以覆盖画布，但仍沿用同一组件和视觉规格，不降级成另一套旧式 Sheet。一个 Inspector 一次只允许一种模式：

### 查看

只展示当前 NodeType、Provider、Connection、验证状态和对象化动作，如“更换连接”。不得出现任何可编辑输入框。

### 选择

只展示当前 NodeType 可使用的配置中心 Connection 候选与“连接新的”。选择新候选前，旧配置保持生效。提供“返回当前配置”。

### 新增/编辑

先选择 Provider，再只展示该 Provider 所需的一套最小草稿表单。标题说明正在连接什么，正文说明“验证成功后保存到配置中心，其他项目可复用”。返回或关闭只丢弃草稿。

### 修复

依次展示：用户结论、仍有效事实、唯一恢复动作、折叠技术详情。不得同时展示无关设置。

以下组合一律禁止：

- 绿色“已连接”卡片下面紧接一套可编辑表单；
- 当前配置、候选列表和新增表单同屏；
- 版本历史或运行日志出现在版本仓库配置；
- 服务器选择、访问地址和运行配置全部同时展开。

节点卡的固定信息层级为：

```text
NodeType
Provider
Connection
配置状态 / 本次运行状态
```

授权不是节点或步骤。未连接、连接正常、授权失效分别通过节点状态和 Inspector 模式表现；Token、密码和私钥永远不出现在节点卡。

## 5. 配置中心复用

- 新增代码平台、版本仓库或服务器时默认保存为可复用连接。
- Connection 显示用户命名、Provider、非敏感端点、最近验证和影响项目；Secret 保存到系统密钥库。
- 节点内只选择或替换当前 NodeInstance 的 Connection 引用，不提供全局删除和批量修改。
- 工作流资源栏和 Inspector 不提供“前往配置中心管理”；全局管理只能从应用首页一级“配置中心”进入。
- 项目的环境变量、业务密钥、域名、线路状态和运行记录不跨项目共享。
- Secret 默认遮盖，不回显、不进入普通日志。

## 6. 表单

- 一次只要求完成当前对象所需字段。
- 先缺值必填，后可选；已有值不重复询问。
- Label 与输入关联；错误紧邻字段；按钮写清结果和对象。
- 前置条件未完成时解释原因并提供入口，不用禁用按钮让用户猜。
- 新服务器优先让用户输入密码；客户端后台生成专用 SSH 密钥并安装，用户不选择私钥文件。

## 7. 状态语言

状态必须是“对象 + 事实”：

| 避免       | 使用                                 |
| ---------- | ------------------------------------ |
| 需要处理   | 运行服务器连接已失效                 |
| 失败       | 新版本启动失败，旧版本仍在线         |
| 待配置     | 还没有选择版本仓库                   |
| 刷新状态   | 检查并继续上线                       |
| TCR 登录   | 连接版本仓库（说明中显示腾讯云 TCR） |
| 配置 Caddy | 设置访问地址                         |

状态同时使用文字、图标和颜色；不能让圆点或颜色独自承担含义。

## 8. 操作层级

- 一个可见状态只突出一个主按钮。
- 次要操作使用次按钮或文字链接；危险操作独立确认。
- 节点就绪时主动作是“更换…”，不是“验证并保存”。
- 选择模式主动作是“使用这个连接”；新增模式主动作是“验证并保存”。
- 长任务说明正在做什么、是否可离开、回来后如何继续。

## 9. 运行记录

运行记录独立于节点抽屉，包含：线路、开始时间、结果、不可变版本、目标服务器和地址。提交、构建号、镜像摘要、原始日志默认折叠。

历史版本重上线从运行记录发起；版本仓库节点只说明“保存在哪里、连接是否可用”。

## 10. 布局与可访问性

- 页面主体只有一个画布滚动区域；抽屉内部使用自身标准滚动。
- 异步更新不重置画布位置、不抢焦点、不整页闪烁。
- 宽屏完整展示四节点；窄窗口横向浏览，节点保持可读尺寸。
- 键盘可切换线路、打开节点库、聚焦节点、打开和关闭 Inspector；关闭后焦点回到原节点。
- 画布平移、缩放、适应视图和内部历史使用 FlowGram 标准行为，不创造 ABCDeploy 私有手势。首版固定线路不把节点拖拽、撤销/重做或增删连线暴露为小白必须操作的功能。
- 图标按钮有包含对象的可访问名称；焦点可见；支持减少动画和强制颜色。

## 11. 页面验收问题

逐屏必须回答：

1. 用户能否看懂代码从本地到服务器的路线？
2. 能否区分节点名称、Provider 和状态？
3. 当前是否只有一个明确下一步？
4. 节点抽屉是否只处于一种模式？
5. 取消修改后旧配置是否仍然有效？
6. 新连接是否明确保存到配置中心并可复用？
7. 运行记录是否与节点配置分离？
8. 能否区分 NodeType、Provider、Connection 和运行状态？
9. 是否优先复用 FlowGram、Coze Studio/Semi Design，而没有自研同类组件？
10. 工作流工作区是否只有一套视觉语言？
