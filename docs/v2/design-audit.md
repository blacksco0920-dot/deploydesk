# ABCDeploy V2 设计审计记录

> 审计对象：`docs/v2/prototype/index.html`<br>
> 审计日期：2026-07-10<br>
> 结论：可进入产品负责人评审，尚不可进入 V2 功能研发。

## 1. 本轮验证范围

已使用真实浏览器逐步点击并验证：

1. 返回用户首页记住已有项目和全局资源。
2. 选择项目与只读识别。
3. CNB 浏览器授权成功回跳状态。
4. 未找到 SSH Key 时自动创建安全连接。
5. 系统生成并默认选中推荐部署方案。
6. 只补充测试域名与正式域名。
7. 部署前展示新增、修改与删除影响。
8. 部署过程刷新后恢复到原阶段。
9. 部署成功后进入项目工作台。
10. DNS 未生效时的解释、人工动作、自动复查和稍后处理。

## 2. 视口验证

| 视口                    | 结果                                                   |
| ----------------------- | ------------------------------------------------------ |
| `1440 x 960`            | 首页、项目工作台和异常页无横向溢出，核心内容完整可见   |
| `1280 x 720`            | 首次部署各步骤无横向溢出，固定底部操作保持可达         |
| `1280 x 720` 部署确认页 | 内容区需要约 26px 纵向滚动，操作栏保持固定，不遮挡内容 |

所有检查页面的 `document.body.scrollWidth` 均等于视口宽度，未发现水平滚动或文本挤出容器。

## 4. 状态恢复验证

在“构建应用镜像”进行中刷新页面，刷新前后均满足：

- 页面标题为“正在部署测试环境”。
- “准备服务器”为已完成。
- “构建应用镜像”为进行中。
- 后续步骤保持等待。

项目、连接状态、域名和当前步骤均从原型检查点恢复。正式实现需改为 SQLite 与系统密钥库，不能照搬 `localStorage`。

## 5. 信息架构验证

- 首次部署使用无侧边栏线性向导，避免把后台菜单误当操作步骤。
- 返回用户首页直接展示最近项目和可复用资源。
- 完成部署后切换到项目级“项目 / 部署 / 环境 / 资源 / 设置”导航。
- 全局资源与项目绑定在视觉上分离，不要求用户为每个项目重新连接 CNB 和服务器。

## 6. 可访问性初检

- 关键输入均有关联标签。
- 图标按钮均有中文可访问名称和 `title`。
- 键盘焦点具有高对比度轮廓。
- 状态不只依赖颜色，同时提供文字和图标。
- 支持 `prefers-reduced-motion`，减少加载动画。
- 主要操作使用结果性文案，不使用含糊的“下一步”或“执行”。

尚未完成屏幕阅读器实机测试、Windows 高对比度模式测试和 200% 缩放测试，进入 P0 发布前必须补齐。

## 7. 已修正问题

| 问题                          | 修正                                               |
| ----------------------------- | -------------------------------------------------- |
| 侧边栏品牌文字对比度不足      | 改为高对比度浅色文字                               |
| 项目工作台仍使用全局导航      | 改为项目级五项导航，并保留“所有项目”返回入口       |
| 首次流程可能暴露 Key 文件选择 | 主路径改为自动发现或生成，手工选择仅放在折叠回退区 |
| 部署错误只显示结果            | DNS 页面拆成已完成、人工动作、自动复查三部分       |

## 8. 未解除门禁

以下内容不能由视觉原型证明：

1. CNB 是否支持公共桌面客户端安全使用的 OAuth 流程。
2. CNB 是否有稳定 API 管理密钥文件与流水线加密变量。
3. Windows、macOS、Linux 的内置 SSH 连接能否通过技术验证。
4. 真实小白用户能否达到首次 staging 独立完成率目标。
5. 服务器 Agent 与受限 SSH 两种部署路径的最终取舍。

这些问题解决前，V2 应用代码继续冻结；允许进行隔离技术验证，但不得把未经证实的流程包装成已完成能力。
