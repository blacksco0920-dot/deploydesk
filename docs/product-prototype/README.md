# ABCDeploy 工作流产品原型

当前原型入口：[workflow-node-config.html](workflow-node-config.html)。

本原型只使用假数据验证产品结构，不连接 CNB、TCR 或服务器，也不证明 FlowGram 已经完成正式集成。

## 本轮验证目标

原型现在包含两个不会同时出现的页面层级：

```text
项目首页：所有项目 / 配置中心｜项目卡片
工作流：线路与节点资源栏｜工作流画布｜节点 Inspector
```

应用级导航只出现在项目列表等外层页面；进入工作流编辑器后隐藏，通过顶部“返回”退出，不能和工作流资源栏同时占据左侧。

首版应用级导航只保留“所有项目”和“配置中心”。不展示没有实际协作能力支撑的“个人空间”，不把项目运行记录放到一级导航，也不在首页常驻展示笼统的本机环境状态；本机路径、服务识别和运行记录都回到具体项目内。

本轮必须可以连续点通以下闭环：

```text
所有项目
→ 添加电脑里的项目
→ 自动识别服务并生成默认线路
→ 依次复用构建、版本仓库和服务器连接
→ 确认开始上线
→ 四个节点依次执行
→ 获得访问地址
→ 返回所有项目并看到“已经上线”
→ 再次进入时主动作变为“更新上线”
```

首页项目卡始终只给一个符合当前状态的动作：未配置时“继续设置”、准备完成时“开始上线”、运行中“查看进度”、成功后“打开工作流”。已经完成的节点和运行状态在页面切换后继续保留。

默认“小白上线模板”自动生成四类基础节点：

```text
代码来源 ──> 版本构建 ──> 版本存储 ──> 部署运行
```

四层概念必须清楚：

- **NodeType**：代码来源、版本构建、版本存储、部署运行；
- **Provider**：本地项目、CNB、腾讯云 TCR、Linux 服务器；
- **Connection**：配置中心保存的账号授权、仓库登录或服务器身份；
- **NodeInstance**：当前线路中对一种 NodeType 的一次使用。

授权不是第五个节点。节点未绑定或授权失效时，在右侧 Inspector 内选择、创建或修复 Connection。

## 可点击内容

1. 从首页添加“客户管理工具”，模拟系统文件夹选择和项目识别；
2. 进入新项目后依次点击三个节点的“使用这个连接”，验证保存后自动推进；
3. 点击“开始上线”，确认目标后查看真实节点顺序的运行投影；
4. 上线完成后查看访问地址，并返回首页确认项目卡已经更新；
5. 再次进入项目，确认状态仍保留且主动作已经变为“更新上线”；
6. 点击画布节点，查看当前 Provider 和 Connection；
7. 点击“更换连接”，从配置中心选择其他连接；
8. 点击“连接新的”，查看 Provider 专属最小表单和“保存到配置中心”；
9. 点击“添加节点”，查看首版四类节点；再次选择“部署运行”会添加服务器分支；
10. 切换左侧线路、拖动画布、拖动节点、缩放和适应视图；
11. 点击顶部历史图标打开运行记录，确认版本历史与节点配置分离。

原型内部仍保留隐藏的状态测试控件，用于自动化覆盖首次配置、授权失效和运行中状态；它不是正式产品组件。正式实现只能根据系统事实更新状态，用户不手动切换。

## 正式实现边界

- 画布、连线、拖拽、缩放、选择、历史记录和快捷键使用 FlowGram；
- 工作区布局、节点库、Inspector、工具栏和表单优先复用 Coze Studio/Semi Design；
- Radix 只作为缺失能力的底层补充，并套用同一工作区视觉令牌；
- ABCDeploy 只开发四类部署节点、连接选择、授权验证、运行状态和底层部署能力；
- 工作流工作区不得混入旧版 ABCDeploy 大卡片、大按钮和旧 Sheet 风格。

## 不是当前规范的文件

同目录的 `app.js`、`styles.css` 和旧 `index.html` 来自历史“发布中心 / 测试版 / 正式版”原型，不代表当前产品模型，等待统一迁入 `docs/archive/`。产品定义以根目录的[产品实施合同](../product-contract.md)为准。
