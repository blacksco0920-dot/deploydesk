# ABCDeploy 错误码

错误码用于准确定位失败环节，不要求用户理解底层命令。客户端同时展示：发生了什么、下一步操作、是否可以重试，以及默认折叠的技术详情。

## 编码规则

```text
AD-<范围>-<编号>
```

| 范围 | 含义 |
| --- | --- |
| `APP` | 客户端通用操作 |
| `LOC` | 本机 Docker 与开发环境 |
| `SSH` | 服务器连接与身份 |
| `SRV` | 服务器运行环境、Caddy 和端口 |
| `REG` | 镜像仓库 |
| `CNB` | CNB 授权、仓库和构建触发 |
| `BLD` | 项目验证和镜像构建 |
| `DEP` | 环境部署 |
| `REL` | 候选版本和镜像摘要核对 |
| `NET` | DNS、HTTPS 和公网访问 |

## 服务器与 Caddy

| 错误码 | 用户看到的原因 | 下一步 |
| --- | --- | --- |
| `AD-SRV-101` | 服务器未安装 Docker | 安装 Docker Engine 与 Compose 插件后重试 |
| `AD-SRV-102` | 当前用户无法使用 Docker | 启动 Docker 并修复用户权限后重试 |
| `AD-SRV-103` | 缺少 Compose 插件 | 安装 `docker-compose-plugin` 后重试 |
| `AD-SRV-201` | 80/443 被未知或多个服务占用 | 保留一个统一反向代理，ABCDeploy 不会自动停止现有服务 |
| `AD-SRV-202` | 已有 Caddy 未开放独立路由目录 | 挂载 `/etc/caddy/sites` 并导入 `sites/*.caddy` |
| `AD-SRV-203` | Caddy 启动失败 | 查看 Docker 日志并重试 |
| `AD-SRV-204` | Caddyfile 校验失败 | 修复现有配置后重试 |
| `AD-SRV-205` | 已保存的 Caddy 适配信息失效 | 重新初始化目标服务器 |
| `AD-SRV-206` | 新路由与现有配置冲突 | 调整重复域名；系统已恢复原路由 |
| `AD-SRV-207` | Caddy 热加载失败 | 检查 Caddy 日志；系统已恢复原路由 |

## 构建与发布

| 错误码 | 用户看到的原因 | 下一步 |
| --- | --- | --- |
| `AD-CNB-101` | CNB 连接已失效 | 重新连接 CNB 后刷新，服务器无需重新初始化 |
| `AD-CNB-103` | 当前令牌权限不足 | 补充 `repo-code:rw`、`repo-manage:rw`、`repo-cnb-trigger:rw` 或 `group-resource:rw` |
| `AD-CNB-201` | 一次性安全配置尚未完成 | 在 CNB 密钥仓库保存测试和生产配置后继续 |
| `AD-CNB-202` | CNB 没有返回构建编号 | 打开最近构建记录核对是否真正启动 |
| `AD-CNB-203` | CNB 未接受构建请求 | 检查仓库与网络后重试 |
| `AD-CNB-204` | 无法读取 CNB 构建状态 | 稍后刷新；不会重复触发构建 |
| `AD-BLD-201` | 项目验证或镜像构建失败 | 查看失败阶段关键日志，修复业务代码后重新部署测试 |
| `AD-DEP-201` | CNB 页面触发的部署未完成 | 打开对应 Tag 的部署记录处理失败阶段 |
| `AD-REL-101` | 无法读取项目发布协议 | 恢复或重新生成 `deploy.yaml` 后核对部署记录 |
| `AD-REL-201` | 无法从服务器读取不可变镜像摘要 | 重新验证服务器连接后刷新，不需要重新构建 |
| `AD-REL-301` | 生产镜像与测试候选不一致 | 保持当前生产版本，重新部署测试并发布新的候选 |
| `AD-NET-201` | 应用已运行，但 DNS 或 HTTPS 尚不可用 | 修正解析或 Caddy 后重新检查，不需要重新构建 |

未知错误使用所属范围的 `299`，例如 `AD-SRV-299`。此时技术详情必须保留脱敏后的原始失败信息，便于支持人员继续定位。
